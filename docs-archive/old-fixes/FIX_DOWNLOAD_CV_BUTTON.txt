## MASALAH:
Ketika klik tombol "Download CV", URL yang dihasilkan salah:
‚ùå api.4leafclover.idhttps://drive.google.com/file/d/1Q0ia3vgMz4TxIdzdO4osGKanLM63c1id/view

URL ini adalah gabungan dari:
- Base URL: `https://api.4leafclover.id`
- Resume URL dari database: `https://drive.google.com/file/d/1Q0ia3vgMz4TxIdzdO4osGKanLM63c1id/view`

## PENYEBAB:
Kode di `HeroSection.tsx` menggunakan:
```tsx
<a href={`${getUploadBaseURL()}${about.resume_url}`}>
```

Ini berasumsi bahwa `resume_url` selalu berupa path relatif seperti `/uploads/cv.pdf`.
Tapi di database, `resume_url` berisi URL Google Drive lengkap!

## SOLUSI:
Update kode untuk mendeteksi apakah `resume_url` sudah URL lengkap atau path relatif:

```tsx
<a 
  href={
    about.resume_url.startsWith('http://') || about.resume_url.startsWith('https://') 
      ? about.resume_url 
      : `${getUploadBaseURL()}${about.resume_url}`
  } 
  target="_blank" 
  rel="noopener noreferrer"
>
```

Logika:
- Jika `resume_url` dimulai dengan `http://` atau `https://` ‚Üí Gunakan langsung (URL eksternal)
- Jika tidak ‚Üí Gabungkan dengan base URL (file lokal di server)

## HASIL:
‚úÖ URL Google Drive: Langsung ke Google Drive
‚úÖ URL lokal (/uploads/cv.pdf): Ke https://api.4leafclover.id/uploads/cv.pdf
