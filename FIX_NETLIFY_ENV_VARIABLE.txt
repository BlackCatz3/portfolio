KENAPA INI PENTING:
-------------------
- Frontend production menggunakan HTTPS (https://4leafclover.id)
- Jika API endpoint pakai HTTP ‚Üí Mixed Content Error
- Browser block HTTP request dari HTTPS page
- Gambar tidak bisa diakses karena request diblokir


CARA FIX DI NETLIFY DASHBOARD:
-------------------------------

STEP 1: Login ke Netlify
-------------------------
1. Buka: https://app.netlify.com
2. Login ke akun Netlify
3. Pilih site "4leafclover.id"


STEP 2: Edit Environment Variable
----------------------------------
1. Klik "Site configuration" (di sidebar kiri)
2. Klik "Environment variables"
3. Cari variable: VITE_API_BASE_URL
4. Klik tombol "Options" (3 titik) ‚Üí "Edit"
5. Ubah value dari:
   ‚ùå http://43.228.213.128/api
   
   Menjadi:
   ‚úÖ https://api.4leafclover.id/api

6. Klik "Save"


STEP 3: Trigger Redeploy
-------------------------
1. Klik "Deploys" (di top menu)
2. Klik "Trigger deploy"
3. Pilih "Clear cache and deploy site"
4. Tunggu deploy selesai (1-2 menit)


ATAU CARA CEPAT (Via CLI):
---------------------------
Jika sudah install Netlify CLI:

netlify env:set VITE_API_BASE_URL "https://api.4leafclover.id/api"
netlify deploy --prod

---------------------------
